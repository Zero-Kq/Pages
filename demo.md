# Markdown 写作规范

本文档记录所有文章必须遵循的 Markdown 写作规范。

---

## 1. Front Matter 字段规范

所有文章必须使用hugo所需要的 YAML 格式 Front Matter。

```yaml
+++
title = ''           # 文章标题
description = ""     # 摘要，用于 SEO 和预览
date = 'YYYY-MM-DD'  # 发布日期
draft = false        # 草稿状态，发布前改为 false
tags = []            # 标签（从白名单选择）
categories = []      # 分类（从白名单选择）
weight = 1           # 排序权重
slug = ""            # 自定义 URL 路径
aliases = []         # 链接别名
series = []          # 系列文章
externalLink = ""    # 外部跳转链接
toc = true           # 自动生成目录
math = false         # 启用数学公式
+++
```

### 必填字段

- `title`
- `date`
- `categories`
- `tags`

### 可选字段

- `description`（建议填写，不超过 150 字）
- `toc`（默认 true）
- `math`（默认 false）
- `weight`、`slug`、`series`、`aliases`（按需使用）

---

## 2. 代码块语言标识

所有代码、命令、配置必须使用代码块，并指定对应语言标识。

| 关键词 | 用途 |
|--------|------|
| ` ```cpp ` | C++ 代码 |
| ` ```python ` | Python 代码 |
| ` ```bash ` | 终端输入命令 |
| ` ```terminal ` | 终端输出结果 |
| ` ```cmake ` | CMake 配置 |
| ` ```yaml ` | YAML 配置文件 |
| ` ```json ` | JSON 数据 |
| ` ```mermaid ` | Mermaid 图表 |

### 代码块书写规则

- 代码块前后各空一行，与正文分隔
- 禁止使用无标识 ` ``` `
- 行内代码仅用于变量、文件名、命令，不用于长文本

---

## 3. 标签（Tags）规范

只能使用以下标签，禁止自创：

- `slam`
- `视觉`
- `雷达`
- `优化`
- `学习笔记`
- `C++`
- `Python`
- `工具`
- `linux`

### 使用规则

- 单篇文章标签数量 ≤ 5 个
- 统一小写，大小写不混用
- 不使用标点、emoji、特殊符号

---

## 4. 分类（Categories）规范

建议分类固定为以下几类，每篇文章最多 1 个分类：

- `SLAM`
- `编程开发`
- `工具配置`
- `学习笔记`
- `Linux`

---

## 5. 标题层级规范

- 文章内部从 `##` 开始，禁止使用 `#`
- 层级必须递进，不跳级：`##` → `###` → `####`
- 标题不使用 emoji、特殊符号、多余标点
- 同一层级标题风格统一
- 同一文章内同一标题下介绍使用同种风格，如:第一个 `参数配置` 使用图表介绍，第二个出现的 `参数配置`也应该使用图表介绍而不是代码块或者文字形式

---

## 6. 参数配置表格

参数说明必须使用表格，禁止使用纯文本罗列。

```markdown
| 参数 | 值 | 说明 |
|------|-----|------|
| xxx  | xxx | xxx |
```

### 表格规则

- 前后各空一行
- 内容简洁，不冗余换行
- 布尔值统一写 `true` / `false`

---

## 7. 常用格式规范

- **粗体**：`**重点内容**`
- *斜体*：`*注释/备注*`
- 行内代码：变量、命令、文件名
- 链接：`[显示文字](链接地址)`
- 图片：`![图片说明](图片路径)`

### 中英文与数字格式

- 英文、数字与中文之间加空格
  - 正确：C++ 代码、Python 脚本、2026 年
- 标点统一使用中文标点（代码与链接除外）

---

## 8. 链接与路径规范

- 站内链接使用相对路径：`[SLAM](/categories/slam/)`
- 站内文章引用优先使用：`{{< ref "文件路径" >}}`
- URL 结尾统一加 `/`
- 外部链接需标注来源，不使用裸链接

---

## 9. 文件命名规范

- 文件名：小写字母 + 数字 + 中横线
  - 示例：`slam-intro.md`、`linux-cmake.md`
- 禁止使用：中文、空格、大写、下划线
- 按分类归档，不全部放在同一目录

---

## 10. 通用排版规范

- 段落之间空一行，不使用多个空行
- 列表统一使用 `-`，不混用 `*` 或数字
- 不重复格式：不同时加粗 + 斜体 + 行内代码
- 长文本分段，避免大段无分割文字
